WHAT "ENTERPRISE REACT" ACTUALLY MEANS
================================================================================

What you have now (Rounds 1-12):
--------------------------------------------------------------------------
  - useState, useEffect with deps, controlled components, derived values
  - Components, props, callback props, lift-state-up, prop drilling
  - Custom hooks (useEmployees, useEmployeeFilter, useSort)
  - React Router v6, protected routes, JWT interceptor
  - Forms with manual validation
  - Pagination, modals, undo, toast feedback
  - Function-identity gotcha + closure-captured IDs (subtler React concepts)


What enterprise React requires that you haven't touched yet:
--------------------------------------------------------------------------
  1. useReducer + Context API
     - State machines and cross-component state without prop drilling.
     - Round 13 — coming next.

  2. Performance hooks (React.memo, useCallback, useMemo)
     - Stop unnecessary re-renders. Critical for tables with many rows.
     - Round 13.

  3. Error boundaries
     - One component's crash shouldn't take down the whole app.
     - Round 13.

  4. TypeScript
     - Non-negotiable for enterprise React.
     - Starts after Round 13.

  5. React Query (TanStack Query)
     - The de facto standard for server state.
     - Would replace your useEffect-fetch pattern in production.

  6. Testing
     - React Testing Library + Jest/Vitest.

  7. Component libraries
     - MUI / Ant Design / Chakra. Nobody styles enterprise apps from scratch.

  8. Form libraries
     - React Hook Form + Zod. Your manual `errors` object is fine for
       learning, never for production.

  9. Next.js (or similar meta-framework)
     - App Router, Server Components, file-based routing.

  10. Architecture patterns
      - Feature folders, compound components, container/presenter split,
        code splitting.
